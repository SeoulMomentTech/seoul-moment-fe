# Gemini CLI 서브 에이전트 시스템 활용 가이드

이 문서는 사용자가 언제든 참고할 수 있도록 작성된 서브 에이전트 활용 가이드입니다. 
(이 문서는 AI가 컨텍스트로 읽지 않도록 `.geminiignore`에 등록되어 있습니다.)

---

## 1. 서브 에이전트 시스템이란?

서브 에이전트 시스템은 메인 AI 에이전트(Tech Lead)가 특정 도메인에 특화된 여러 "전문가 에이전트"를 도구처럼 호출하여 복잡한 작업을 분산 처리하는 구조입니다.

### 🔥 주요 장점
1. **토큰 소모 최적화 (비용 절감 및 속도 향상)**
   - 일반적인 작업 방식에서는 파일 탐색, 코드 수정, 에러 해결 등 여러 번의 상호작용(턴)이 메인 세션에 누적되어 토큰 소모가 기하급수적으로 늘어납니다.
   - 서브 에이전트에게 위임하면, 서브 에이전트가 격리된 공간에서 작업을 마친 뒤 **"결과 요약본 하나"**만 메인 세션으로 반환합니다. 메인 컨텍스트가 매우 가볍게 유지됩니다.
2. **역할 분담과 전문성 향상**
   - 웹(Next.js), 어드민(Vite), 아키텍처(구조 설계) 등 각 영역의 전문 지식을 가진 에이전트가 해당 영역의 작업을 전담하여 퀄리티가 높아집니다.
3. **복잡한 작업의 병렬/순차 처리**
   - 한 번의 지시로 다수의 프로젝트(예: `apps/web`과 `apps/admin` 동시 수정) 작업을 효율적으로 조율할 수 있습니다.

---

## 2. 서브 에이전트 호출 방법

특별한 명령어나 문법을 배울 필요 없이 **자연어로 메인 에이전트에게 지시**하면 됩니다.

### A. 자동 위임 (기본 방식)
작업 내용만 명확하게 지시하면, 메인 에이전트가 판단하여 적절한 서브 에이전트를 조용히 백그라운드에서 호출합니다.
> "apps/web의 헤더 컴포넌트 여백을 수정해줘."
> "apps/admin 대시보드 페이지에 새로운 버튼을 추가해줘."

### B. 명시적 호출
특정 에이전트를 지정하고 싶을 때 에이전트의 이름을 직접 언급합니다.
> "이 작업은 `web-specialist`에게 넘겨서 진행해줘."
> "`senior-architect`를 호출해서 현재 폴더 구조 리팩토링 계획을 작성해줘."

---

## 3. Tech Lead 오케스트레이션 프롬프트 팁 (추천)

규모가 큰 작업이거나, 프론트/백오피스 양쪽 모두 수정이 필요한 작업일 경우 **메인 에이전트를 리더(오케스트레이터)로 명확히 지정**하는 것이 좋습니다.

### 💡 복사해서 쓰기 좋은 프롬프트 예시

**1. 대규모 작업 시 파이프라인 구성 요청 (가장 추천)**
> "새로운 '사용자 리뷰' 기능을 개발해야 해. **Tech Lead로서 이 작업을 분석하고 계획을 세워줘.** 
> 먼저 `senior-architect`를 시켜서 데이터 흐름과 상태 관리 구조를 설계하게 하고, 그 구조가 확정되면 `web-specialist`와 `admin-specialist`에게 각각의 컴포넌트 구현을 지시해서 전체 작업을 완료해줘."

**2. 기존 코드 분석 및 리팩토링 위임**
> "장바구니 결제 로직 코드가 너무 복잡해. **이건 규모가 큰 작업이니까 니가 오케스트레이터 역할을 맡아서 처리해.**
> `senior-architect`를 호출해 개선안을 도출한 다음, 그 결과에 따라 `web-specialist`를 시켜서 실제 Next.js 코드를 안전하게 리팩토링해줘."

**3. 버그 탐색 및 수정 위임**
> "현재 어드민 페이지에서 날짜 필터를 적용하면 에러가 발생해. 
> 메인 세션이 길어지면 안 되니까, `admin-specialist`를 호출해서 해당 에러의 원인을 찾고 코드를 수정하게 한 다음 **최종 결과만 나한테 요약해서 보고해줘.**"
